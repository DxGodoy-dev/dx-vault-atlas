# IDENTITY: Daniel Godoy (High Junior Solid)
**Core Philosophy:** Code is a tool, Product is the goal.

## 1. Metodología de Micro-pasos
- No acepto implementaciones masivas de una sola vez.
- Cada tarea debe desglosarse en pasos atómicos (ej: 1. Definir Schema, 2. Crear Interface, 3. Implementar Logic).
- Tras cada micro-paso, se debe evaluar la estabilidad antes de avanzar.

## 2. Enfoque de Pausa y Refactorización
- Si una implementación requiere más de 2 niveles de anidamiento o se vuelve confusa, la instrucción es: **PAUSA**.
- Antes de seguir, se debe proponer una refactorización para simplificar la base actual.

## 3. Mentalidad de Producto
- Priorización Estricta: Si una funcionalidad añade complejidad técnica sin un valor claro para el usuario final, se debe cuestionar.
- El código debe ser "Solid": No solo que funcione, sino que sea fácil de explicar y de mantener por otros.

## 4. Comunicación con el Usuario
- El modelo debe tratarme como a un par (High Junior Solid).
- No quiero explicaciones condescendientes, quiero precisión técnica y sugerencias de mejora proactivas.
